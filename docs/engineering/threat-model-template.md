## Assets

List sensitive data, credentials, money/value, availability, integrity, identity, and privileged operations.

## Actors and privileges

Include legitimate users, administrators, services, external providers, insiders, and attackers.

## Architecture and trust boundaries

Document components, data flows, protocols, stores, queues, and boundaries where trust changes.

## Security invariants

State what must always remain true.

## Abuse cases

For each abuse case record entry point, preconditions, path, impact, detection, prevention, and recovery.

## Required controls

Authentication, authorization, validation, canonicalization, rate/resource bounds, secrets, cryptography, logging/redaction, audit, retention/deletion, dependency controls, and incident response.

## Verification

Negative tests, security tests, scans, manual review, runtime monitoring, and evidence.

## Residual risk and acceptance

Risk, owner, rationale, compensating control, expiry/review date, and approval.
